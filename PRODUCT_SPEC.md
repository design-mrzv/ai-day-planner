## 1. Суть продукту

AI-планер дня. Юзер вводить текстом все що в голові — AI перетворює хаос на структуровані задачі й план на сьогодні.

**Робоча назва:** AI Day Planner  
**Референс:** Todoist (стиль і UX), Any.do (простота)  
**Стек:** Claude Code → GitHub → Vercel  
**AI модель для парсингу:** ~~claude-haiku-4-5~~ → **Gemini Flash (Google AI Studio)** — тимчасова заміна через технічні проблеми з оплатою Anthropic Console (не пройшла жодна картка, безкоштовні кредити не активувались). Промпт і JSON-контракт лишаються ідентичними, змінюється тільки провайдер API.  
**API ключ:** зберігається в env-змінній `GEMINI_API_KEY` на Vercel (ніколи не в коді). Якщо оплату Anthropic вдасться вирішити до дедлайну — повертаємось на `ANTHROPIC_API_KEY` і claude-haiku-4-5.

## 2. Структура задачі

Що AI витягує з тексту юзера:

| Поле | Тип | Примітка |
|------|-----|----------|
| Назва | string | Обов'язкове |
| Пріоритет | high / medium / low | AI визначає автоматично |
| Дедлайн | today / tomorrow / date | AI визначає з контексту |
| Час | string або null | Якщо згаданий в тексті |
| Лейбл | work / personal | AI визначає автоматично |
| Description | string або null | Необов'язкове, тільки якщо є деталі в тексті |

**Приклад інпуту:**
> "Завтра здати звіт, зустріч о 14:00, купити продукти після роботи"

**Приклад аутпуту AI (JSON):**
```json
[
  {
    "title": "Здати звіт",
    "priority": "high",
    "deadline": "tomorrow",
    "time": null,
    "label": "work",
    "description": null
  },
  {
    "title": "Зустріч",
    "priority": "high",
    "deadline": "today",
    "time": "14:00",
    "label": "work",
    "description": null
  },
  {
    "title": "Купити продукти",
    "priority": "low",
    "deadline": "today",
    "time": null,
    "label": "personal",
    "description": "після роботи"
  }
]
```

## 3. Екрани та навігація

### Екран 0 — Welcome (тільки перший запуск)
- Показується один раз
- Зберігати в localStorage: `onboarding_done: true`
- Контент: назва продукту + одне речення суті + кнопка "Почати"
- Копі кнопки: **"Почати"**
- Слоган: *"Розкажи все, що в голові. Ми складемо план на день"*

### Екран 1 — Today (головний екран)
- Великий заголовок "Today"
- Поточна дата під заголовком
- Список задач на сьогодні
- Якщо список порожній — порожній стан з текстом та ілюстрацією
- FAB кнопка "+" внизу справа — пульсує при першому відкритті після onboarding
- Іконка Settings вгорі справа

**Порожній стан:**
- Ілюстрація або іконка
- Текст: *"Що плануєш сьогодні? Натисни + і розкажи все що в голові"*
- Кнопка "+" пульсує щоб привернути увагу

**Картка задачі показує:**
- Чекбокс (виконано / не виконано)
- Назва задачі
- Час (якщо є)
- Лейбл (work / personal) — кольоровий тег
- Пріоритет — індикатор кольором (червоний / жовтий / сірий)
- Дедлайн (якщо не сьогодні)
- Тап на задачу → розгортає Description (якщо є)

### Екран 2 — Capture (bottom sheet)
Відкривається по кнопці "+"

- Великий текстовий інпут: *"Що в голові?..."*
- Підказка сірим: *"Пиши все підряд, ми в цьому розберемось"*
- Кнопка "Обробити" (активна коли є текст)
- Стан завантаження: *"Ми будуємо твій план..."* з анімацією

**Флоу за замовчуванням (без Inbox):**
Capture → AI обробляє → задачі одразу в Today → bottom sheet закривається

**Флоу з Inbox увімкненим (Settings):**
Capture → AI обробляє → Inbox екран → юзер підтверджує або редагує → Today

### Екран 3 — Inbox (тільки якщо увімкнено в Settings)
- Заголовок: *"Ось що ми знайшли"*
- Список розпарсених задач для перегляду
- Можливість редагувати кожну задачу
- Кнопка "Додати все в Today"
- Кнопка "Назад" (повернутись до Capture)

### Екран 4 — Settings
- Один тогл: **"Переглядати задачі перед додаванням"** (вмикає Inbox)
- За замовчуванням: вимкнено

## 4. Основні флоу

### Флоу A — Перший запуск
1. Welcome screen
2. Тап "Почати"
3. Today (порожній стан, "+" пульсує)
4. Тап "+"
5. Capture bottom sheet
6. Вводить текст → "Обробити"
7. AI парсить → задачі в Today
8. Bottom sheet закривається

### Флоу B — Повторний запуск
1. Одразу Today зі збереженими задачами
2. Тап "+" → Capture → Today

### Флоу C — З Inbox увімкненим
1. Тап "+" → Capture → "Обробити"
2. AI парсить → Inbox екран
3. Юзер переглядає / редагує
4. "Додати все в Today" → Today

## 5. Технічні вимоги

- **Mobile-first** — весь сценарій має проходити однією рукою
- **localStorage** — зберігати задачі та стан onboarding (без бекенду в MVP)
- **API ключ** — тільки через env-змінну на Vercel (`GEMINI_API_KEY`), ніколи в коді браузера
- **Деплой** — GitHub + Vercel, кожен push автоматично оновлює сайт
- **Перевірка AI відповіді** — парсити JSON з відповіді моделі, обробляти помилки

## 7. Для дизайнера

**Референси:** Todoist iOS, Things 3  
**Стиль:** мінімалістичний, чистий, mobile-first  
**Пріоритети дизайну:**
1. Порожній стан Today — має виглядати як запрошення, не як баг
2. Capture bottom sheet — великий зручний текстовий інпут
3. Картка задачі — вся інформація читається за 1 секунду
4. Welcome screen — один меседж, жодного зайвого тексту

**Кольори пріоритетів:**
- High → червоний
- Medium → жовтий / помаранчевий
- Low → сірий

**Кольори лейблів:**
- Work → синій
- Personal → зелений

## 8. Для програміста

**Головне правило:** спочатку працює парсинг тексту → задачі в Today. Все інше потім.

**Порядок реалізації:**
1. Деплой порожнього проекту на Vercel
2. Today екран з хардкодженими задачами (перевірка верстки)
3. Capture bottom sheet + виклик API
4. Парсинг відповіді AI → відображення задач
5. localStorage (зберігання між сесіями)
6. Welcome / onboarding
7. Settings + Inbox (якщо залишився час)

**Prompt для AI парсингу:**
```
Ти — асистент планування. Юзер описав свої задачі вільним текстом.
Витягни всі задачі і поверни їх як JSON масив.

Для кожної задачі визнач:
- title: назва задачі (коротко, дієслово + об'єкт)
- priority: "high" | "medium" | "low"
- deadline: "today" | "tomorrow" | "YYYY-MM-DD" | null
- time: "HH:MM" | null
- label: "work" | "personal"
- description: додаткові деталі | null

Повертай ТІЛЬКИ валідний JSON масив. Без пояснень, без markdown.

Текст юзера: {USER_INPUT}
```

**Примітка про AI-провайдера:** через непройдену оплату Anthropic Console (див. розділ 1) використовуємо Gemini Flash через Google AI Studio (`GEMINI_API_KEY`, безкоштовно, без картки). Модель — `gemini-2.5-flash` або `gemini-2.5-flash-lite`, обидві покривають structured JSON output. Промпт вище лишається без змін — міняється тільки виклик API. Якщо Anthropic оплату вирішимо — легко повернутись, бо контракт (JSON-схема) не залежить від провайдера.
