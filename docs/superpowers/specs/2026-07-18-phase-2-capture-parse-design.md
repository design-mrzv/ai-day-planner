# Phase 2 — Today (hardcoded) → Capture → AI Parse → Today

> Джерело вимог: `PRODUCT_SPEC.md`, розділ 8, кроки 2-4.
> Мета: перший наскрізний сценарій "текст → задачі на екрані" працює. Без localStorage, Settings, Inbox.
>
> **Оновлення 2026-07-19:** AI-провайдер змінено з Anthropic (`claude-haiku-4-5`) на Gemini Flash (`gemini-2.5-flash`, Google AI Studio) через непройдену оплату Anthropic Console — див. `PRODUCT_SPEC.md` §1. Промпт і JSON-контракт не змінились, ключ тепер `GEMINI_API_KEY`.

## Архітектура

- `src/app/page.tsx` — Today екран: заголовок "Today", поточна дата, список задач (`useState<Task[]>`, старт з 1-2 хардкоджених задач), FAB "+", порожній стан.
- `src/components/task-card.tsx` — картка задачі: чекбокс, назва, час (якщо є), лейбл-тег (work/personal), індикатор пріоритету кольором, дедлайн (якщо не сьогодні), тап розгортає description.
- `src/components/capture-sheet.tsx` — bottom sheet на базі `vaul` Drawer: великий textarea з плейсхолдером "Що в голові?...", підказка сірим "Пиши все підряд, ми в цьому розберемось", кнопка "Обробити" (неактивна при порожньому тексті), loading-стан "Ми будуємо твій план..." з анімацією, error-стан.
- `src/app/api/parse/route.ts` — Next.js Route Handler (server-side): приймає `{ text: string }`, викликає Google GenAI SDK (`gemini-2.5-flash`, ключ з `GEMINI_API_KEY` env) з промптом із розділу 8 спеку, парсить JSON з відповіді, повертає масив задач або помилку.
- `src/lib/types.ts` — тип `Task` (title, priority, deadline, time, label, description) + `id`, `done` для UI-стану.

**Потік даних:** Capture textarea → `POST /api/parse` → масив задач у JSON → `setTasks(prev => [...prev, ...parsed])` у `page.tsx` → Drawer закривається → нові картки з'являються в Today.

**UI-компоненти:** shadcn/ui + `vaul` для Drawer/bottom sheet (accessible примітиви, нативний swipe-to-close для мобільного UX).

**Стан задач:** тримається в `useState` на рівні `page.tsx`. Втрачається при refresh — прийнятно для цієї фази, localStorage додається пізніше (крок 5 з розділу 8).

## Обробка помилок

Усі кейси показують один і той самий людський текст у Capture sheet: **"Не вдалося обробити, спробуй ще раз"** + кнопка повторити. Sheet не закривається, введений текст юзера не втрачається.

Кейси, що покриті цим повідомленням:
- Anthropic повернув невалідний/непарситься JSON (`JSON.parse` кидає помилку на сервері → 422)
- Мережева помилка або 5xx від Anthropic API
- AI повернув порожній масив задач (юзер щось вводив, але задач не розпізнано — теж трактуємо як помилку, а не як "успішно нічого не додано")
- Markdown-обгортка навколо JSON (```json ... ```) — стрипається на сервері перед парсингом як запобіжник, хоча промпт це забороняє

## Стилі (розділ 7 спеку)

- Пріоритет (колір-індикатор на картці): high → червоний, medium → жовтий/помаранчевий, low → сірий
- Лейбл (кольоровий тег): work → синій, personal → зелений
- Порожній стан Today: іконка/ілюстрація + текст-запрошення "Що плануєш сьогодні? Натисни + і розкажи все що в голові" — виглядає як запрошення, не як баг
- Capture sheet: великий зручний textarea — головний фокус екрана

## Тестування (ручне, для цієї фази)

Без автотестів — за порядком реалізації з розділу 8 перевіряємо вручну на кожному кроці:
1. `npm run dev` → верстка Today з хардкодженими задачами коректна
2. Тап "+" → Capture sheet відкривається, swipe-to-close працює
3. Ввід прикладу з розділу 2 спеку ("Завтра здати звіт, зустріч о 14:00, купити продукти після роботи") → 3 задачі з'являються в Today з правильними полями
4. Симуляція помилки (порожня AI-відповідь / зламаний route handler) → людське повідомлення + retry, текст не втрачається

## Що свідомо НЕ робимо в цій фазі

localStorage, Welcome/onboarding, Settings, Inbox-екран — за явним запитом користувача, це наступні кроки з розділу 8.
